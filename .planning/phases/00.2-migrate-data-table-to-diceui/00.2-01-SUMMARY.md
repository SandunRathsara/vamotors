# Phase 00.2 Plan 01: DataTableShell onStateChange + 4 Consumer Migrations Summary

DataTableShell exports `DataTableState` and fires `onStateChange` callback from `useDataTable` URL state; vehicles, purchases, customers, and users tables migrated to consume it, eliminating duplicate `useQueryStates` calls in each consumer.

## What Was Built

**DataTableShell changes (components/shared/data-table-shell.tsx):**
- Exported `DataTableState` interface with `page`, `perPage`, `sortBy`, `sortDir` fields
- Added `onStateChange?: (state: DataTableState) => void` prop
- Added `enableAdvancedFilter: false` to `useDataTable` call (explicit per D-05)
- Added `React.useEffect` that fires `onStateChange` when `pagination` or `sorting` state changes from `table.getState()`
- Converts 0-indexed `pageIndex` to 1-indexed `page` for consumers
- All backward-compat props kept (`isFiltered`, `onClearFilters`, `searchPlaceholder`)

**Consumer migration pattern (applied to 4 tables):**
- Removed `useQueryStates`, `parseAsInteger`, `parseAsString` imports from nuqs
- Added `React.useState<DataTableState>` with sensible defaults
- Updated `useEntityQuery` to read from `tableState` instead of individual nuqs params
- `pageCount` calculation uses `tableState.perPage`
- Removed `isFiltered`, `onClearFilters`, `searchPlaceholder` from DataTableShell JSX
- Added `onStateChange={setTableState}` to DataTableShell

**Column meta.label additions:**
- `vehicles-columns.tsx`: makeModel, year, listedPrice, costBasis, purchaseDate (5 columns)
- `purchases-columns.tsx`: purchaseDate, vehicle, purchasePrice, costBasis (4 columns)
- `customers-columns.tsx`: fullName (1 column)
- `users-columns.tsx`: name, role, status, lastLogin (4 columns)
