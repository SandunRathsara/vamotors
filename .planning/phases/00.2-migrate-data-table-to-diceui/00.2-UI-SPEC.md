# Phase 0.2 — UI Design Contract

> Visual and interaction contract for the DiceUI data-table migration. Generated by gsd-ui-researcher.
>
> **Nature of this phase:** Infrastructure swap only. No new screens, no new user-visible UI surfaces. The design contract governs what MUST NOT change (all visual tokens, layout, and copy are inherited from Phase 0.1), plus the specific interaction contracts for the filter/sort UI that becomes functional after wiring.

## Spacing Scale

Declared values (multiples of 4). All inherited from Tailwind v4 defaults — no changes in this phase.

| Token | Value | Usage in data-table context |
|-------|-------|----------------------------|
| xs | 4px | Icon gaps within toolbar buttons, cell icon-to-text gap |
| sm | 8px | Toolbar button padding, filter tag padding |
| md | 16px | Table cell horizontal padding, toolbar horizontal padding |
| lg | 24px | Section padding above/below table |
| xl | 32px | Page-level side padding |
| 2xl | 48px | Not used in table context |
| 3xl | 64px | Not used in table context |

Exceptions: Table empty-state minimum height is 300px (`min-h-[300px]`), enforced in `DataTableShell`. This is a fixed pixel value, not a spacing scale value — it ensures the empty state is visually prominent. Do not remove it.

## Typography

All sizes use the system sans-serif stack via `--font-sans`. Inherited from globals.css — no phase-specific overrides.

| Role | Size | Weight | Line Height | Used in data-table context |
|------|------|--------|-------------|---------------------------|
| Body | 14px (0.875rem) | 400 (regular) | 1.5 | Table cell content, filter input text, pagination labels |
| Label | 14px (0.875rem) | 500 (medium) | 1.4 | Column header text via `DataTableColumnHeader`, filter field names in filter list |
| Heading | 18px (1.125rem) | 600 (semibold) | 1.2 | Empty-state heading (`text-lg font-semibold` — already in `EmptyState` component) |
| Small | 12px (0.75rem) | 400 (regular) | 1.4 | Pagination item count text (`text-sm` on muted), filter operator badges |

Note: "text-sm" in Tailwind v4 defaults to 0.875rem (14px). Heading uses `text-lg` = 1.125rem (18px). These are pre-existing in the EmptyState component — do not change them.

## Color

All colors are OKLCH CSS custom properties from the radix-vega preset. No hex values are used directly — reference tokens only.

| Role | Token | OKLCH (light mode) | Usage in data-table context |
|------|-------|-------------------|----------------------------|
| Dominant (60%) | `--background` | `oklch(1 0 0)` (white) | Table row background, page background |
| Secondary (30%) | `--card` / `--sidebar` | `oklch(1 0 0)` / `oklch(0.985 0 0)` | Table header row (`bg-muted/50`), toolbar background |

Accent reserved for: active column sort direction indicator, selected filter field highlight in `DataTableFilterList`, active state ring on Filter and Sort toolbar buttons only. Do NOT apply primary/accent color to regular table row hover states — use `--muted` for row hover.

## Interaction Contracts

Interaction contracts for UI behaviors that become active after this migration. These were non-functional before (buttons rendered, but had no effect).

### Filter List (`DataTableFilterList`)

| State | Behavior | Visual |
|-------|----------|--------|
| No filters active | Filter button shows no badge | `<DataTableFilterList>` renders "Add filter" trigger only |
| 1+ filters active | Filter button shows count badge | Each filter row shows: field selector, operator selector, value input, remove (X) button |
| Filter field selector | Dropdown lists all columns with `enableColumnFilter: true` | Columns shown by `meta.label` value — must be set on all filterable columns |
| Filter operator | Per-variant operator list from `config/data-table.ts` | "contains", "is", "is not", etc. depending on `meta.variant` |
| Filter value | Renders the correct filter input per `meta.variant` | text → Input; select/multiSelect → faceted popover; date/dateRange → calendar |
| Filter reorder | Drag handle on each filter row — uses `@dnd-kit/sortable` | Drag icon visible on hover |
| Join operator | AND/OR toggle above filter rows | Toggle between "Match all" and "Match any" |
| Clear all filters | "Clear filters" link in filter list footer | Resets `filters` and `joinOperator` URL params |
| Filtered empty state | When filter active + 0 rows | EmptyState shows "No results match your filters" + "Clear filters" button (already in `EmptyState` component) |

### Sort List (`DataTableSortList`)

| State | Behavior | Visual |
|-------|----------|--------|
| No sorts active | Sort button shows no badge | "Add sort" trigger only |
| 1+ sorts active | Sort button shows count badge | Each sort row: column label, ASC/DESC toggle, remove button |
| Sort column selector | Lists all columns with `enableSorting: true` | Columns shown by `meta.label` — must be set on all sortable columns |
| Sort reorder | Drag handle — multi-column sort order settable | Drag icon visible on hover |
| Column header click | Cycles: none → asc → desc → none | Sort indicator arrow in `DataTableColumnHeader` |

### Pagination

| State | Behavior |
|-------|----------|
| URL param name | `perPage` (not `pageSize`) — consumers must read `perPage` from URL for API calls |
| Default page size | 20 rows (set via `initialState` in `DataTableShell`) |
| Page size options | 10, 20, 30, 50 — standard diceui pagination defaults |
| URL param for page | `page` (integer, 1-indexed) |

### URL State

| Param | Format | Owner |
|-------|--------|-------|
| `page` | Integer (e.g., `1`) | `useDataTable` via `DataTableShell` |
| `perPage` | Integer (e.g., `20`) | `useDataTable` via `DataTableShell` |
| `sort` | JSON array (e.g., `[{"id":"makeModel","desc":false}]`) | `useDataTable` via `DataTableShell` |
| `filters` | JSON array of filter objects | `DataTableFilterList` directly |
| `joinOperator` | `"and"` or `"or"` | `DataTableFilterList` directly |

Consumers read these same params (via `useQueryState`) to drive their API calls. The `sort` JSON param must be parsed client-side to extract `sortBy` and `sortDir` for the mock Route Handlers (which accept separate string params).

## What Must NOT Change

The following are locked from the existing Phase 0.1 design. This migration must produce zero visual regressions against these.

| Contract | Locked Value | Where Enforced |
|----------|-------------|----------------|
| Table row background | `--background` (white) | `data-table.tsx` — do not add alternate row striping |
| Table header background | `bg-muted/50` | `data-table.tsx` — do not change header color |
| Row hover state | `hover:bg-muted/50` | `data-table.tsx` — do not change to primary color |
| Cell padding | `p-4` (16px) | `data-table.tsx` — do not change |
| Empty state icon | `Inbox` from lucide-react | `empty-state.tsx` — do not swap icon |
| Empty state icon size | `h-10 w-10` (40px) | `empty-state.tsx` — do not change |
| Toolbar layout | `DataTableAdvancedToolbar` flex row | `data-table-shell.tsx` — toolbar children remain left-aligned; filter+sort buttons remain right-aligned |
| Pagination position | Below table, full width | `data-table.tsx` — do not move |
| Border radius | `--radius: 0.45rem` → `rounded-md` | globals.css — do not override per-component |
