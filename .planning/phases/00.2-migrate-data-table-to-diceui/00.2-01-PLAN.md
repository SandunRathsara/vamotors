Modify `DataTableShell` in `components/shared/data-table-shell.tsx` to expose the useDataTable hook's URL state to consumers via an `onStateChange` callback. This is the key architectural change that prevents duplicate URL state (Pitfall 1 from RESEARCH.md).

**Changes to DataTableShellProps interface:**

Add these new props:
```typescript
/** Callback fired when useDataTable URL state changes. Consumer reads this to drive API calls. */
onStateChange?: (state: DataTableState) => void
```

Add this new type export above the interface:
```typescript
export interface DataTableState {
  page: number
  perPage: number
  sortBy: string
  sortDir: "asc" | "desc"
}
```

**Changes to the component body:**

1. The `useDataTable` call stays as-is but add `enableAdvancedFilter: false` explicitly (per D-05, adopting diceui's hook â€” it already owns URL state via nuqs internally for `page`, `perPage`, `sort`).

2. After the `useDataTable` call, extract the current pagination and sorting state from `table.getState()` and fire `onStateChange` via a `React.useEffect`:

```typescript
const { pagination, sorting } = table.getState()

React.useEffect(() => {
  if (!onStateChange) return
  const sortEntry = sorting[0]
  onStateChange({
    page: pagination.pageIndex + 1,  // convert 0-indexed to 1-indexed
    perPage: pagination.pageSize,
    sortBy: sortEntry?.id ?? "",
    sortDir: sortEntry?.desc ? "desc" : "asc",
  })
}, [pagination.pageIndex, pagination.pageSize, sorting, onStateChange])
```

3. Keep `isFiltered` and `onClearFilters` props for backward compat during migration. They will be removed in Plan 03 cleanup.

4. Keep `searchPlaceholder` prop (unused but harmless during migration).

**Do NOT change:** The JSX structure, DataTableAdvancedToolbar, DataTableFilterList, DataTableSortList, EmptyState rendering, or the `min-h-[300px]` empty state container. Per UI-SPEC "What Must NOT Change" section.
