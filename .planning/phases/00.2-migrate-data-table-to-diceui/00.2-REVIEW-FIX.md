### WR-03: User Deletion Targets Collection Endpoint Instead of Resource Endpoint

**Files modified:** `components/users/users-columns.tsx`
**Commit:** be40268

## Skipped Issues

### WR-04: Module-Scope Fixture Lookup Maps Will Silently Return Wrong Data After API Migration

**File:** `components/customers/customers-columns.tsx:19-30`, `components/repairs/repairs-columns.tsx:22-23`, `components/sales/sales-columns.tsx:24-25`
**Reason:** No actionable fix at this time. The review explicitly states this is "an acceptable tradeoff during the mock-data phase" and provides guidance for "when real API is wired." The fixture lookup maps work correctly with the current mock data. This will need to be addressed when migrating to real API endpoints.
**Original issue:** Three column files build lookup Maps at module scope from imported fixture arrays. These maps are computed once when the module is first imported and never updated. When real API data replaces fixtures, these columns will silently show stale data.
