## Fixed Issues

### WR-01: Destructive test endpoint gated by a single env var with no defense-in-depth

**Files modified:** `app/api/test/reset/route.ts`
**Commit:** 582fa15
**Applied fix:** Added a hard 404 whenever `VERCEL_ENV === "production"` so no env var combination can open the gate on a Vercel production deploy. Kept the existing `NODE_ENV === "production"` + `TEST_RESET_ENABLED` check for local `next start` compatibility. Added an optional signed header check: when `TEST_RESET_SECRET` is set, the request must present a matching `x-test-reset-secret` header. The handler now reads the `Request` object to inspect headers.

### WR-02: Reset handler ignores the request; no origin/auth check and no SECURITY TODO breadcrumb

**Files modified:** `app/api/test/reset/route.ts`
**Commit:** 582fa15
**Applied fix:** Added an explicit `SECURITY TODO(phase-1)` comment listing the remaining hardening items (origin allow-list, rate limit, audit log, mandatory secret) before the Prisma swap. The handler now accepts `request: Request` and inspects it for the signed header, so the endpoint is no longer request-blind. Committed together with WR-01 since both findings touch the same file and are conceptually a single hardening pass.

### WR-03: tests/auth.setup.ts navigates to `/` which may not exist, risking cascading setup failure

**Files modified:** `tests/auth.setup.ts`
**Commit:** ace2efa
**Applied fix:** Replaced the `page`-fixture version with a `browser`-fixture version that creates a fresh context, writes an empty `storageState`, and closes the context — no navigation at all. This decouples the setup project from any known-good URL, so `api` and `e2e` projects no longer cascade-fail if `/` is missing or redirects. Added a comment explaining why the navigation was removed.

### WR-04: toCents can produce `-0` for inputs like "-0" or "-0.00"

**Files modified:** `lib/money.ts`, `lib/money.test.ts`
**Commit:** a9a14f1
**Applied fix:** Normalized the zero case in `toCents`: if the arithmetic result is `=== 0`, return positive `0` explicitly. This avoids the `Object.is(-0, 0) === false` asymmetry that would leak into fixture diffs and sort stability. Added three regression tests covering `"-0"`, `"-0.00"`, and `"-0.0"` using `Object.is(..., 0)`. Full `lib/money.test.ts` suite: 11 tests passing.
