## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Renamed `app/api/__test__/reset/route.ts` → `app/api/test/reset/route.ts`**

- **Found during:** Task 3 verification (`pnpm build` did not compile the route).
- **Issue:** Next.js App Router treats any folder whose name starts with an underscore (`_folderName`) as a **private folder** and excludes it from the routing system (verified via Context7 against `nextjs_app_getting-started_project-structure`). The planned path `app/api/__test__/reset/route.ts` starts with `_`, so Next.js silently skipped it during `pnpm build`, meaning `POST /api/__test__/reset` would always 404 — making the reset contract completely non-functional, not just production-guarded.
- **Fix:** Renamed the directory to `app/api/test/reset/route.ts`. The public URL becomes `POST /api/test/reset`. Path prefix is still "un-pretty" and "discoverable only by reading the test harness" per T-0.3-03 threat model intent. All internal logic (NODE_ENV 404 guard, resetAll call, runtime pin) is verbatim from RESEARCH.md Pattern 4.
- **Verified:** Ran `NODE_ENV=production pnpm start -p 3457` + `curl -s -o /dev/null -w "%{http_code}" -X POST http://localhost:3457/api/test/reset` → returned **404**. Production guard confirmed working.
- **Files modified:** `app/api/test/reset/route.ts` (moved, not rewritten)
- **Commit:** `1965793`

**2. [Rule 2 - Missing critical functionality] `passWithNoTests: true` in vitest.config.ts**

- **Found during:** `just check` post-task verification.
- **Issue:** Vitest 4.x exits with code 1 when the include glob matches zero test files. Plan 01 deliberately ships zero tests (Plan 02 lands them), so `just test-unit` and therefore `just check` always fail on a fresh Plan-01-only checkout. Plan 01's stated `done` criterion is "zero test files exist yet in Plan 01 so vitest reports 0 tests passed cleanly" — this is a correctness requirement for the `check` recipe to be usable immediately.
- **Fix:** Added `passWithNoTests: true` to `vitest.config.ts` `test` block with an inline comment explaining the Plan 02 removal window.
- **Verified:** `just test-unit` now exits 0 with "No test files found, exiting with code 0".
- **Commit:** `9d922c8`
