Prove the Plan 01 harness actually works by landing the canonical first test at every tier — unit, component, API, and E2E — each exercising real Phase 0.1/0.2 code with no internal mocks.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1/3 — Bug] Plan 01's `NODE_ENV=production` 404 gate made `/api/test/reset` unreachable under Playwright's webServer**

- **Found during:** Task 2, first run of `playwright test --project=api`. The `beforeEach` reset call returned non-OK and all API assertions failed.
- **Root cause:** Playwright's `webServer.command` is `pnpm build && pnpm start`. `pnpm start` runs `next start`, which forces `NODE_ENV=production`. Plan 01's reset handler short-circuited to 404 whenever `NODE_ENV === "production"`, so every test run hit 404 on the reset. The Plan 01 smoke test had confirmed the 404 on a direct `NODE_ENV=production pnpm start` invocation but never ran an end-to-end Playwright test against that environment, so the contradiction slipped through.
- **Fix:** Extended the gate so it accepts a `TEST_RESET_ENABLED=1` env override, and wired `playwright.config.ts` `webServer.env` to set `TEST_RESET_ENABLED: '1'`. Real Vercel deployments still see `NODE_ENV=production` without `TEST_RESET_ENABLED`, so the 404 is still the default posture. T-0.3-01 mitigation survives.
- **Files modified:** `app/api/test/reset/route.ts`, `playwright.config.ts`.
- **Commit:** `f209064`.
- **Verified:** After the fix, `PORT=3457 pnpm exec playwright test --project=api tests/api/vehicles.api.spec.ts` → 4 passed.

**2. [Rule 3 — Blocker] Chromium v1217 not in cache, no network to download**

- **Found during:** First Playwright run reported `browserType.launch: Executable doesn't exist at .../chromium_headless_shell-1217/...`. `pnpm exec playwright install chromium` failed with ENOTFOUND on `cdn.playwright.dev`.
- **Root cause:** This worktree's node_modules has `@playwright/test@1.59.1` which bundles chromium v1217, but the host's `~/Library/Caches/ms-playwright/` only had v1208 from a prior project. No network access in this environment.
- **Fix:** Symlinked the cached v1208 directories into the v1217 slots:
  ```
  ~/Library/Caches/ms-playwright/chromium-1217 -> chromium-1208
  ~/Library/Caches/ms-playwright/chromium_headless_shell-1217 -> chromium_headless_shell-1208
  ```
  Both sub-versions are ABI-compatible for the basic Chrome DevTools Protocol usage Playwright needs here; all 4 tests pass. Future machines still need `pnpm exec playwright install chromium` — this fix is local to the current worktree cache only, not version-controlled.
- **Files modified:** None (filesystem symlinks outside repo).
- **Commit:** N/A (not a repo change).
- **Follow-up:** When network is available, run `pnpm exec playwright install chromium` to populate the real v1217 directory and delete the symlinks.

**3. [Rule 3 — Blocker] Port 3000 already in use**

- **Found during:** First Playwright run — port 3000 occupied by the developer's running `pnpm dev` session.
- **Fix:** Used `PORT=3457` env var on every Playwright invocation. `playwright.config.ts` already reads `process.env.PORT` and builds `baseURL` accordingly, so no config change was needed.
- **Files modified:** None.
- **Commit:** N/A.

Typecheck, format-check, and test-unit segments are all green. Plan 02's own new files (the 5 test files + 2 modified) are individually lint-clean.
- **`just ci` green end-to-end** — was not executed because `just check` fails on lint first. The lint debt is Plan 01's documented deferred item; un-blocking Plan 02's `just ci` requires a separate pass through `deferred-items.md` (candidate for a pre-Phase-1 quick task).

Plan 02's own tier-specific `just test-*` recipes are all green, which is the substantive "platform is verified" bar the plan sets. The lint debt is orthogonal.

### Out-of-Scope Discoveries (deferred)

- **`process.env.PORT` not respected by `just test-*` recipes automatically** — developers must remember to export `PORT=<free port>` if :3000 is taken. Not worth a Justfile change for Phase 0.3 (single-user dev environment); Phase 1 CI runs on a clean port anyway. Logged here for visibility.
- **Playwright browser cache version drift** — the symlink fallback above works but is brittle. Phase 1 CI should pin the exact chromium version via a `PLAYWRIGHT_BROWSERS_PATH` or a `postinstall` hook that runs `playwright install`.
