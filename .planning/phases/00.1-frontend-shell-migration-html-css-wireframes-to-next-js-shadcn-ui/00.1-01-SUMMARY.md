### @originui Registry Probe Result

**Status: BROKEN — fallback active**

Probe command: `pnpm dlx shadcn@latest add "https://originui.com/r/input-phone"`
Result: `Unexpected token '<', "<!DOCTYPE "... is not valid JSON`

The registry at `originui.com` returns an HTML redirect rather than the expected JSON component registry response. This confirms the UI-SPEC finding (originui acquired by COSS/coss.com, now points to an incompatible Base UI library). All downstream plans must use shadcn official patterns for:
- Phone input: hand-rolled `react-phone-number-input` wrapper
- Multi-select/combobox: shadcn `Command` + `Popover` composite pattern
- Date range picker: shadcn `Calendar` + `Popover` + `react-day-picker`
