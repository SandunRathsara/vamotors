## What Was Built

### Task 1 — Sales list + Route Handlers + Record Sale dialog

- `GET /api/sales` — pagination, saleType/status filters, date range filtering (post-query on full set), sortBy/sortDir
- `POST /api/sales` — creates sale with auto-status by sale type
- `GET/PATCH /api/sales/[id]` — detail fetch and partial update, `await params` pattern
- `SalesColumns` — 9 columns: #, Date, Vehicle (linked), Sale Type (badge), Customer (linked), Sale Price, Profit/Loss (green/red), Status (StatusBadge), Actions
- `SalesTable` — `useEntityQuery('sales')`, nuqs params (page, saleType, status, dateFrom, dateTo), empty state from UI-SPEC
- `RecordSaleDialog` — form with vehicle/customer/saleType/price/payment method/date/mileage; `useEntityMutation` with toast "Sale recorded. Vehicle marked as Sold."
- `SalesPage` — `"use client"` for dialog state, PageHeader with "Record Sale" CTA

### Task 2 — Sale detail with timeline

- `SaleDetailPage` — server component, `await params`, renders `<SaleDetailView id={id} />`
- `SaleDetailView` — `useEntityDetail('sales')`, breadcrumb "Sales / Sale #id", two-column layout (main + 280px sidebar)
- Payment breakdown varies by sale type:
  - **Cash**: Sale Price + Payment Method
  - **Advance**: Advance Amount, %, Agreed Price, Expiry Date, Next Steps panel
  - **LeaseFinance**: Down Payment, Finance Amount, Finance Company, DO Status/Document link, disabled "Download Invoice" button (Phase 5 placeholder)
  - **TradeIn**: Sale Price, Trade-In Value, computed Cash Balance, Traded Vehicle link
- Quick Stats card: Sale Type badge, Sale Price, Status, Date, Profit/Loss
- Customer Info card with link to `/customers/[id]`
- Vehicle Info card with link to `/vehicles/[id]`
- `SaleTimeline` — vertical timeline with circle markers (filled=first, outline=last) and connecting border-l line

### Task 3 — Repairs list + Send for Repair dialog

- `GET /api/repairs` — pagination, status/vendorId filters, sortBy/sortDir
- `POST /api/repairs` — creates repair with status "InProgress"
- `PATCH /api/repairs` — updates repair by id in body
- `RepairsColumns` — 9 columns: #, Vehicle, Vendor, Repair Request (truncated to 60 chars), Date Sent, Date Returned, Cost (or "—"), Status, Actions (View Vehicle)
- `RepairsTable` — `useEntityQuery('repairs')`, nuqs params, empty state from UI-SPEC
- `SendRepairDialog` — Vehicle select, Vendor select (RepairVendor type only), Repair Request textarea, Date Sent; `useEntityMutation` with toast "Vehicle sent for repair. Status updated to In Repair."
- `RepairsPage` — `"use client"` for dialog state, PageHeader with "Send for Repair" CTA
