**2. [Rule 1 - Bug] CommandMenu controlled-open toggle**
- **Found during:** Task 1
- **Issue:** When `Header` controls the open state via props, the `useEffect` Cmd+K handler used functional updater `(prev) => !prev` which doesn't work when `setOpen` is an external `onOpenChange` callback (expects a plain boolean).
- **Fix:** Changed to `setOpen(!open)` with `[open, setOpen]` in the effect dependency array.
- **Files:** `components/layout/command-menu.tsx`
- **Commit:** 3daa19a
