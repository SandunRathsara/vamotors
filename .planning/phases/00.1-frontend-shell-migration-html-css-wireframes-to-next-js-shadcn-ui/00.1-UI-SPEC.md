## Spacing Scale

Declared values (multiples of 4 only). The preset uses Tailwind v4 `@theme` with a standard 4-point base. All spacing in components must use tokens from this scale.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, tight inline padding |
| sm | 8px | Compact element spacing, table cell padding (horizontal) |
| md | 16px | Default element spacing, card body padding, form field gaps |
| lg | 24px | Section padding, sidebar group gaps, dialog padding |
| xl | 32px | Layout gaps between major panels |
| 2xl | 48px | Major section breaks, page header bottom margin |
| 3xl | 64px | Page-level top padding (main content area above first section) |

Exceptions:
- Touch targets for icon-only buttons (sidebar collapse, topbar icons): minimum **44px** height to meet WCAG 2.1 AA minimum (use `h-11 w-11` = 44px in Tailwind).
- Sidebar width: `--sidebar-width` token from preset (do not override — accept preset value, typically `16rem / 256px`).
- Sidebar collapsed width: `--sidebar-width-icon` token from preset (typically `3rem / 48px`).

## Typography

Font: **Inter** (self-hosted via `next/font/google`). All sizes in rem/px pairs below reference the Tailwind v4 type scale. Use exactly these four roles — no ad-hoc font sizes outside this contract.

| Role | Size | Tailwind Class | Weight | Weight Class | Line Height |
|------|------|---------------|--------|-------------|-------------|
| Body | 14px / 0.875rem | `text-sm` | Regular 400 | `font-normal` | 1.5 |
| Label | 12px / 0.75rem | `text-xs` | Regular 400 | `font-normal` | 1.4 |
| Heading | 20px / 1.25rem | `text-xl` | Semibold 600 | `font-semibold` | 1.25 |
| Display | 28px / 1.75rem | `text-2xl` | Semibold 600 | `font-semibold` | 1.2 |

Usage notes:
- **Body (14px/400/1.5):** table cell text, form input values, paragraph descriptions, toast messages, sidebar nav labels.
- **Label (12px/400/1.4):** table column headers, form field labels, badge text, status chips, breadcrumb segments. Labels are visually distinguished from body by size (12px vs 14px) — weight differentiation is not needed and would introduce a third weight.
- **Heading (20px/600/1.25):** page section headings, card titles, dialog titles, tab panel headings.
- **Display (28px/600/1.2):** page-level titles (the `<h1>` at the top of each route), dashboard KPI numbers.

Two weights only: **400 (Regular)** and **600 (Semibold)**. Do not use 300, 500, 700, or 800 anywhere in Phase 0.1.

## Color

Source: the `radix-vega` preset (`b2BoX602a`) produces OKLCH tokens in `app/globals.css`. All color references below are semantic token names from that file. Do NOT use raw hex values in components — always use the CSS custom property. The exact hex equivalents below are approximate for reference only.

| Role | CSS Token | Approx. Value | Usage |
|------|-----------|---------------|-------|
| Dominant (60%) | `--background` | zinc-950 dark / zinc-50 light | Page background, main content area background |
| Secondary (30%) | `--card` / `--sidebar-background` | zinc-900 dark / zinc-100 light | Cards, sidebar panel, nav background, dialogs, popovers, dropdowns |
| Accent (10%) | `--primary` | green/teal hue (preset green primary) | See accent reserved list below |
| Destructive | `--destructive` | red (preset destructive) | Destructive actions only — see destructive list below |

**60/30/10 split clarification:**
- 60% `--background`: the main content scrolling area, page shell behind the sidebar.
- 30% `--card` / `--sidebar-background`: sidebar panel, topbar, all Card components, Sheet/Drawer overlays, Dialog backgrounds.
- 10% `--primary`: interactive focus states, active sidebar item highlight, primary CTA buttons, progress indicators.

**Accent (`--primary`) reserved for these specific elements only:**

1. Primary CTA buttons (the single call-to-action per screen — e.g., "Record Purchase", "Save Changes", "Add Customer").
2. Active/selected sidebar navigation item highlight (background + text color).
3. Focus ring on all interactive elements (`ring-primary` / `outline-primary`).
4. Badge on the Approvals sidebar item (unread count).
5. Progress/loading spinner or skeleton shimmer accent line.
6. Link text within body copy (if any links appear in descriptions or empty states).

No other elements may use `--primary`. Navigation hover states use `--accent` (the preset's secondary interactive surface token, not `--primary`).

**Destructive (`--destructive`) reserved for:**
1. Delete / Remove buttons and their confirmation dialogs.
2. Error state text and error toast background.
3. "Cancel Advance (no refund)" and "Cancel Advance (with refund)" action buttons on the Approvals screen.
4. Form validation error messages.

## Interaction States

Required interactive states for all interactive components. These must be implemented — no placeholder or TODO states.

| State | Requirement |
|-------|------------|
| Loading | `loading.tsx` per route segment renders a Skeleton shimmer. Tables show a skeleton of the column count with 5–8 placeholder rows. Cards show shimmer blocks matching approximate content height. |
| Empty | Custom empty state per list screen (see Copywriting Contract). No blank whitespace. |
| Error | `error.tsx` at root and per route group. Query error state renders an inline error card with problem + action. HTTP 400/404 from Route Handlers propagates to TanStack Query error branch. |
| Not Found | `app/not-found.tsx` — branded, includes VA Motors name, links back to `/dashboard`. Triggered by `notFound()` for unknown `[id]` params. |
| Hover | Sidebar items: `bg-accent` on hover. Table rows: `bg-accent/50` on hover. Buttons: follow shadcn variant defaults. |
| Focus | All interactive elements have a visible `ring-2 ring-primary` focus ring. No `outline-none` without a visible replacement. |
| Disabled | Buttons disabled during in-flight mutations (use TanStack Query `isPending`). "Download Invoice" button on sale-detail is always disabled (Phase 5 placeholder). |
| Toast success | Sonner toast — green-tinted, 4s auto-dismiss, top-right position. Shows entity name + action (e.g., "Vehicle added to inventory"). |
| Toast error | Sonner toast — red-tinted, persists until dismissed (no auto-dismiss on error). Shows error reason + "Try again" implicit. |

## Screen-Level Notes

Critical visual contracts per screen type that the executor must not infer.

### App Shell (all `/(app)/*` routes)

- Sidebar: shadcn `<Sidebar>` with five `<SidebarGroup>` sections: **Main** (Dashboard, Vehicles, Purchases, Sales, Repairs), **Management** (Customers, Third Parties, Reports, Approvals), **Finance** (Reconciliation, Cash Flow), **Brokerage** (Lease Deals, Comparison Tool, Rate Sheets, Dispatch), **System** (Settings, Users & Roles).
- Approvals item renders a `<Badge>` with unread count sourced from `/api/approvals?status=pending`. Badge uses `--primary` accent color.
- Topbar: positioned as a `<Header>` sibling above `<SidebarInset>`. Contains: global search `<Input>` (triggers `<Command>` palette on focus or Ctrl+K/Cmd+K), notification bell `<Button>` with `<Popover>`, theme toggle `<Button>` (sun/moon icon, switches dark/light), avatar `<DropdownMenu>`.
- **Icon-only topbar button accessibility:** All three icon-only topbar actions must carry explicit `aria-label` attributes:
  - Notification bell `<Button>`: `aria-label="Notifications"`
  - Theme toggle `<Button>`: `aria-label="Toggle theme"`
  - Avatar `<DropdownMenu>` trigger `<Button>`: `aria-label="User menu"`
- Mobile: sidebar collapses to an off-canvas `<Sheet>` (Vaul drawer). Hamburger icon in topbar triggers it.

### Login (`/login`)

- Built from shadcn `login-04` block.
- Static form: email + password fields, "Remember me" checkbox.
- "Sign In" button: any input accepted — clicks route to `/dashboard` with no validation beyond HTML `required` attribute.
- No error state (static mock — D-27).
- Outside the `(app)` route group — no sidebar, no topbar.

### Dashboard (`/(app)/dashboard`)

- Built from shadcn `dashboard-01` block.
- KPI cards (Display size 28px for the number, Label 12px for the descriptor): In Stock count, In Repair count, Active Advances count, Recent Sales (last 7 days count).
- Quick Actions row: three `<Button>` — "Record Purchase" (→ `/purchases`), "Record Sale" (→ `/sales/new` or opens dialog), "Send for Repair" (→ `/repairs` with new-repair dialog).
- Bar chart (recharts via shadcn Chart): "Units Sold per Month" — 12 months of mock data. Green primary color for bars.
- Recent Activity Feed: last 8 activity entries from mock fixture. Timestamps formatted with `date-fns`.

### List Screens (vehicles, sales, repairs, customers, third-parties, approvals, all reports, cash-flow, lease-deals, lease-dispatch, lease-reconciliation, users)

- All use `@diceui/data-table`.
- URL state managed by `nuqs` (page, pageSize, sortBy, sortDir, q, status, dateFrom, dateTo per screen).
- Default `pageSize`: 20 rows.
- Pagination: previous/next + page number display showing "Showing X–Y of Z".
- Search: debounced 300ms before firing query key update.
- Column headers that sort: show up/down caret on hover; show solid caret with direction on active sort.
- Row actions: `<DropdownMenu>` with at minimum "View" (→ detail route) and context-specific actions (Edit, Delete/Archive where applicable).

### Detail Screens (vehicle-detail, sale-detail, customer-detail, third-party-detail, lease-deal-detail)

- Breadcrumb at top: e.g., "Vehicles / Toyota Corolla 2022" using shadcn `Breadcrumb`.
- Tabs for multi-section content using shadcn `Tabs`.
- Right-side or top Quick Stats panel where wireframe shows it (vehicle-detail, sale-detail).
- File gallery (vehicle-detail, lease-deal-detail): `<Carousel>` with `<Progress>` during local upload.

### Purchases (`/(app)/purchases`)

- shadcn `Tabs` for purchase type: Cash Purchase | Lease Settlement | Brand-New.
- Each tab is an independent form wired to TanStack Form v1 + Zod schema.
- "Record Purchase" is the primary CTA — green `<Button>` at bottom-right of each form.

### Settings (`/(app)/settings`)

- shadcn form sections (no full-page tabs — use `<Card>` per settings group or a left-nav settings layout).
- Advance Payment Tiers table: editable inline using `@diceui/sortable` if reorder is needed; otherwise a plain `<Table>` with edit-in-place cells.

Components have been replaced with shadcn official alternatives — see Component Inventory section.

**Executor instruction for future third-party vetting:**

Before adding any component from @diceui (optional), @magicui, or any other third-party registry, run:


Scan output for: `fetch(`, `XMLHttpRequest`, `navigator.sendBeacon`, `process.env`, `eval(`, `Function(`, `new Function`, dynamic imports from external URLs, obfuscated variable names. If any flag is found, stop and report to the developer before proceeding.
