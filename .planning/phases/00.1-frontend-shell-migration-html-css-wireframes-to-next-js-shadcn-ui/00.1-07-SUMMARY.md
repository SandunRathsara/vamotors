## What Was Built

### Customers (Task 1)

- `GET /api/customers` — paginated list, full-text search via `q` param covers name/NIC/phone
- `GET /api/customers/[id]` — single customer + joined sales with vehicle info, 404 for unknown id
- `POST /api/customers` — create customer
- CustomersTable with `nuqs` URL state, debounced search input
- customers-columns: Name (link to detail), NIC/Passport, Phone, Email, Address (truncated), Vehicles (count from sales fixtures), Last Purchase date, Actions dropdown
- AddCustomerDialog: required Full Name, NIC/Passport, Address, Phone; optional Calling Name, Email, Driving Licence — toast "Customer added successfully."
- CustomerDetailView: Breadcrumb "Customers / {fullName}", info card (8 fields), purchase history table with sale type, date, price, status — each row links to `/sales/[saleId]`

### Third Parties (Task 2)

- `GET /api/third-parties` — paginated, filterable by `type` param (Supplier/RepairVendor/FinanceCompany)
- `GET /api/third-parties/[id]` — entity + historyData array populated based on type (vehicles for Supplier, repairs for RepairVendor, lease sales for FinanceCompany)
- `PATCH /api/third-parties/[id]` — for archive (sets isActive=false)
- ThirdPartiesTable: shadcn Tabs (All/Suppliers/Repair Vendors/Finance Companies) + DataTableShell + Archive confirm dialog in table
- third-parties-columns: Name, Type badge, Category, Total Volume, Contacts (count with hover tooltip showing names/phones), Status, Last Activity, Actions (View/Archive)
- AddThirdPartyDialog: conditional Finance Company fields (Commission Rate, Processing Path), dynamic Contact Persons array (add/remove)
- ThirdPartyDetailView: type-based branching renders 3 distinct layouts — Supplier shows purchase history, RepairVendor shows repair history, FinanceCompany shows lease deals. Archive button in header with ConfirmDialog using canonical copy from UI-SPEC.
