## What Was Built

### Reports Hub (`/reports`)
- `ReportHubCards`: responsive grid (1/2/3 cols) of 7 clickable Card components with Lucide icons
- Hub page uses `PageHeader` + `ReportHubCards`

### ReportTable Component
- Generic `ReportTable<TData>` wrapping `DataTableShell` with: breadcrumb (Reports / {title}), back link, date range filter (dateFrom/dateTo), nuqs URL state, `useEntityQuery`
- dateRangeFilter prop: false on Stock (snapshot), true on all others

### 7 Report Route Handlers
Each at `/api/reports/[type]`:
- **stock**: derives from `getStockReport()`, daysSincePurchase calculated
- **sales**: derives from `getSalesReport(dateFrom, dateTo)`, joined with vehicles/customers
- **purchases**: derives from vehicleFixtures, joined with thirdPartiesFixtures, additionalCosts summed
- **repairs**: derives from `getRepairCostReport(dateFrom, dateTo)`, joined with vehicles/vendors
- **advances**: filters salesFixtures by saleType=Advance, joined with vehicles/customers
- **profit**: derives from `getProfitLossReport`, joins repair costs and additional costs for full breakdown
- **audit**: returns activityFixtures with date filtering

### 7 Report Pages
Each page defines inline `ColumnDef[]` with `CurrencyDisplay` for monetary values and `StatusBadge` for status fields.

### Cash Flow (`/cash-flow`)
- Route handler: filters by date + type, computes totalInflow/totalOutflow/netPosition from filtered set
- Page: 3 summary cards (inflow green, outflow red, net position color-coded green/positive / red/negative)
- Transaction table: Date, Type badge, Description, Reference, Amount (arrow icon + green/red color)
- Loading: 3 skeleton cards + SkeletonTable
