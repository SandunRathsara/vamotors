    PageHeader "Reports" + `<ReportHubCards />`.

    **3. Create `components/reports/report-table.tsx`:**
    Generic reusable report table component. Props: `title: string`, `columns: ColumnDef<any>[]`, `endpoint: string`, `queryKey: string`, `dateRangeFilter?: boolean`.
    - Renders PageHeader with Breadcrumb "Reports / {title}" + back link to /reports

    - Empty state: "No data for this period" / "Adjust the date range or filters to find records."

    **4. Create 7 Route Handlers** (all follow same pattern):
    Each `app/api/reports/[type]/route.ts` exports GET with `injectLatency()`, reads dateFrom/dateTo/page/pageSize/sortBy/sortDir params, derives report data from the mock stores (joining across entities as needed), returns paginated response.

    - **stock** (`app/api/reports/stock/route.ts`): queries vehiclesStore, columns: Vehicle (make+model), Status, Listed Price, Days In Stock (calculated from purchaseDate), Availability, Purchase Date.
    - **sales** (`app/api/reports/sales/route.ts`): queries salesStore joined with vehicles/customers, columns: Date, Vehicle, Sale Type, Customer, Sale Price, Cost Basis, Profit, Margin %.
    - **purchases** (`app/api/reports/purchases/route.ts`): derives from vehiclesStore, columns: Date, Vehicle, Purchase Type, Supplier, Purchase Price, Additional Costs, Total Cost.
    - **repairs** (`app/api/reports/repairs/route.ts`): queries repairsStore joined with vehicles/thirdParties, columns: Vehicle, Garage, Repair Request, Date Sent, Date Returned, Days Taken, Cost.
    - **advances** (`app/api/reports/advances/route.ts`): queries salesStore where saleType=Advance, columns: Vehicle, Customer, Advance Amount, Advance %, Placed Date, Expiry Date, Status, Resolution.
    - **profit** (`app/api/reports/profit/route.ts`): queries sold vehicles with full cost breakdown, columns: Vehicle, Purchase Price, Additional Costs, Repair Costs, Total Cost, Sale Price, Profit/Loss, Margin %.
    - **audit** (`app/api/reports/audit/route.ts`): returns mock audit entries from activity fixtures, columns: Timestamp, User, Action, Entity, Record, Changes, Reason.

    **5. Create 7 report pages** (each reuses `<ReportTable>`):
    Each `app/(app)/reports/[type]/page.tsx` renders `<ReportTable>` with report-specific columns and endpoint. Each has a corresponding `loading.tsx` with SkeletonTable.

    Column definitions for each report are defined inline in the page file (since each has unique columns). Use StatusBadge and CurrencyDisplay where appropriate.
